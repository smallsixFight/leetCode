### 不同路径（Unique Paths）
[原题来源：力扣（LeetCode）](https://leetcode-cn.com/problems/unique-paths/)

#### 题目
一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为“Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为“Finish”）。

问总共有多少条不同的路径？

![robot_maze.png](https://i.loli.net/2020/03/25/A1gvcCJmSwtVK9E.png)

例如，上图是一个7 x 3 的网格。有多少可能的路径？

示例 1：
```
输入: m = 3, n = 2
输出: 3
解释:
从左上角开始，总共有 3 条路径可以到达右下角。
1. 向右 -> 向右 -> 向下
2. 向右 -> 向下 -> 向右
3. 向下 -> 向右 -> 向右
```
示例 2：
```
输入: m = 7, n = 3
输出: 28
```

提示：

- `1 <= m, n <= 100`
- 题目数据保证答案小于等于`2 * 10 ^ 9`

#### 思路一
这道题可以发现到达当前位置的路径数量为从起点到达该位置左侧和上侧的位置的路径数之和，那么就能想到自底向上和动态规划。

#### 思路二
这是在题解看到的思路：机器人到达终点向下走几步和向右走几步都是确定的，总的步数为`m+n-2`，向下走的步数为`m-1`。

将向下走表述为 0，向右走表述为 1，那么只需要找到`m-1`个 0 在`m+n-2`中有多少个排列组合。

不过这个思路需要对越界进行处理。

#### 伪代码
```
UNIQUE-PATHS(m. n)
    dp = [m-1][n-1]int
    for i = 0 to n -1
        dp[0][i] = 1
    for i = 0 to m -1
        dp[j][0] = 1
    for i = 1 to m -1
        for j = 1 to n -1
            dp[i][j] = dp[i-1][j] + dp[i][j-1]
    return dp[m-1][n-1]
```
